## Overview 

It is an Customer support Portal 
- User is Author 
    1 Can View published Books and the detials and Status of the Book They have Published
    2 Can Raise Ticket if they have any query related to any type of issue they are facing in thier book

- Admin 
    1 Views all Ticket/Queries raised  by the author for their books
    2 Provide any response and llok and provide any personal solution they want
    3 Bookleaf admin portal have different page then teh user

## Tech Stack 

- Backend : Django | Django Rest Framework
- Frontend : React + vite
- Database : PostgreSQL (Supabase)
- LLM : Gemma-4 (from gemini API)
- Backend Deployment : Render
- Frontend Deployment : Vercel
- Package Mangers : cPython, bun
